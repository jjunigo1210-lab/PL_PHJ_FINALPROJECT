# PL_PHJ_FINALPROJECT
English Vocabulary Study Bot

# 1. 프로그램 소개

이 프로그램은 영어 단어 학습 프로그램이다.  
중고등학생 시절에 영어 단어를 외울 때 내가 정말 이 단어를 외웠는지와 같이, 메타인지가 제대로 되지 않아 어려움을 느꼈었다. 
그 기억을 기반으로 영어 단어를 직접 가져와서 퀴즈형식으로 외우는 것에 도움을 줄 수 있는 프로그램을 기획하였다 .

사용자는 영어 단어와 뜻을 직접 추가하고, 퀴즈를 통해 학습하며,  
프로그램은 그 결과를 파일로 저장해 다음 복습에 활용한다.  
복잡한 기능보다는 실제로 꾸준히 사용할 수 있는 구조를 만드는 데 중점을 두었다.


# 2. 기획 의도

영어 공부를 하다 보면 단어를 외웠다고 생각해도 금방 잊어버리는 경우가 많다.  
또 어떤 단어를 언제 다시 봐야 하는지 스스로 판단하기 어렵다는 점도 불편하게 느껴졌다.

이러한 경험을 바탕으로, 단어를 단순히 저장하는 것을 넘어서  
이 단어를 얼마나 잘 외우고 있는지를 확인할 수 있는 프로그램을 만들고자 했다.


# 3. 개발 과정

먼저 단어와 뜻을 저장하고 불러오는 기능을 구현하려고 하였다.  
단어 학습은 하루로 끝나지 않기에, 프로그램을 종료해도 단어장이 유지되도록 CSV 파일을 사용했다.

다음으로 단어별 학습 상태를 관리하기 위해 JSON 파일을 사용했다.  
각 단어에 대해 복습 날짜와 정답 횟수, 오답 횟수를 기록하여 정확한 점수 체계를 보며 공부할 수 있도록 했다.

그 이후 퀴즈 기능을 추가하였다.  
뜻을 보여주고 영어 단어를 직접 입력하게 하여, 정답 여부를 바로 확인할 수 있도록 구성하였다. 

퀴즈 결과에 따라 복습 날짜가 달라지도록 하여,  
자주 틀리는 단어는 더 자주 보게 되고, 잘 아는 단어는 자연스럽게 복습 간격이 늘어나도록 하였다.

마지막으로 학습 통계와 단어 관리 기능을 추가하였다.  
전체 정답률과 자주 틀린 단어를 확인할 수 있도록 하였고,  
잘못 추가한 단어를 삭제하거나 전체 데이터를 초기화할 수 있도록 정리하였다. 
추가적으로 프로그램을 실행하는 동안 중복단어를 추가했을 때, 그대로 중복단어를 출력하는 것에 불편함을 느껴 중복 단어는 처음부터 제외하도록 했다.


# 4. 프로그램 기능 설명

단어 추가 기능(add)을 통해 영어 단어와 뜻을 저장할 수 있다.  
이미 등록된 단어를 다시 입력할 경우 중복이라는 안내 메시지가 출력되며, 단어는 추가되지 않는다.

단어 목록 기능(list)은 현재 단어장에 저장된 모든 단어를 확인할 수 있도록 한다.

오늘 복습 기능(today)은 복습 날짜가 오늘 이전인 단어만 선별하여 보여준다.  
이를 통해 어떤 단어를 우선적으로 공부해야 할지 쉽게 알 수 있다.

퀴즈 기능(quiz)은 뜻을 보고 영어 단어를 입력하는 방식으로 진행된다.  
정답과 오답에 따라 학습 기록이 저장되며, 이후 복습 일정에 반영된다.

통계 기능(stats)을 통해 전체 정답률과 많이 틀린 단어를 확인할 수 있다.

단어 삭제 기능(remove)은 잘못 입력한(오타, 외울 필요 없는) 단어를 단어장과 학습 기록에서 함께 제거한다.  
초기화 기능(reset)은 단어장과 학습 기록을 모두 초기 상태로 되돌린다.

우선, add 명령어를 이용하여 영어 단어를 입력 후, 뜻을 입력하여 단어장에 저장할 수 있다.
입력된 단어는 words. csv 파일에 저장되고 프로그램을 종료해도 단어장이 유지되도록 하였다.

 add명령어 없이 단어를 입력하거나 정해진 명령어 이외의 것을 입력하면, 알 수 없는 명령어로 판단하고 unknown command 메시지를 출력하도록 했다. 

 list 명령어를 사용하면 현재 단어장에 저장된 모든 단어가 번호, 뜻과 함께 출력된다.
 이를 통해 단어장 전체를 한눈에 확인할 수 있다.

 today 명령어를 통해 오늘 quiz를 진행해야할 단어들(복습 날짜가 오늘 이전인 단어)를 선별하여 출력한다.
 처음 단어를 추가한 경우에도 복습 대상으로 본다.

 quiz 명령어를 입력하면 뜻을 보고 영단어를 입력하는 퀴즈가 시작된다.
 이는 단어장 전체로 진행하며 한 번 정답을 맞춘 단어는 복습 단어로 바꾸어 오늘은 다시 퀴즈를 했을 때 나오지 않도록 했다.(이후에 좀 더 효율적으로 발전한 형태로 바꿔서 진행해보고 싶다.) 정답을 입력하면 correct 메시지, 오답을 입력하면 실제 정답과 함께 wrong 메시지를 출력한다. 각 퀴즈의 결과는 attempts.csv 파일에 저장되며, 정답과 오답 여부에 따라 복습 날짜를 조정한다.

 stats 명령어를 이용하면 지금까지의 학습 결과를 확인할 수 있다. 전체 정답률과 가장 많이 오답이 많은 단어를 출력한다. 예시에서는 go 를 틀려, 가장 많이 틀린 단어로 표시하였다.

 remove 명령어를 통해 특정 단어를 단어장에서 삭제할 수 있다. 위 예시에서 go를 삭제하여 이후 단어 목록에서 나타나지 않도록 했다.

 이미 단어장에 있는 단어를 추가하려고 하면 중복이라는 메시지가 뜨며 불필요하게 단어를 추가하는 것을 막았다.

 모든 단어의 복습 날짜가 오지 않았을 때, today 명령어를 입력하면 오늘 복습할 단어가 없다고 출력한다.

 reset 명령어를 사용하여 csv 파일(단어장)과 학습 기록(attempts) 파일을 모두 초기화할 수 있도록 하였다. 또한 실수 혹은 호기심으로 데이터가 삭제되는 것을 막기 위해 YES를 입력해야지 초기화가 비로소 진행되도록 하였다.

 마지막으로, exit 명령어를 입력하면 간단한 종료 메시지를 응원과 함께 띄우며 프로그램을 종료하도록 했다.




# 7. 마지막으로

이번 프로젝트는 지난번에 만들었던 챗봇을 그냥 두지 말고, 끝까지 한번 완성해보자는 생각에서 시작했다. 처음에는 단어를 저장하고 다시 불러오는 기능만 있으면 충분할 것 같았다. 그런데 막상 직접 실행해서 써보니 생각보다 불편한 부분이 많이 보였다. 단어를 같은 걸 여러 번 넣었을 때 계속 저장된다든지, 명령어를 잘못 쳤을 때 아무 반응이 없는 상황 같은 것들이 있었다. 이런 부분은 코드 자체보다도 사용 흐름에서 더 신경 써야 하는 문제라는 걸 느꼈다. 그래서 단어 중복을 막는 처리랑 잘못된 입력에 대한 안내를 하나씩 추가했고, 초기화 기능도 바로 실행되지 않게 확인 문구를 넣게 되었다. 

데이터 저장 방식도 처음에는 단순하게 생각했다. 그냥 파일에 저장하면 되는 줄 알았는데, 막상 구현하다 보니 데이터 성격이 서로 다르다는 게 보였다. 단어랑 뜻은 그대로 남아야 하고, 복습 날짜나 정답 횟수 같은 건 계속 바뀐다. 그래서 words.csv에는 단어 정보를 넣고, progress.json에는 복습 상태를 따로 관리했다. 퀴즈 기록은 attempts.csv에 남겨서 나중에 통계를 낼 수 있게 했다. 퀴즈 결과에 따라 날짜를 계산하는 부분도 해보면서, 조건문이 실제로 어떻게 쓰이는지 조금은 알게된 것 같다.

개발하면서 가장 신경 쓴 건 기능을 하나 추가했을 때 다른 부분이 망가지지 않게 하는 거였다. 예를 들어 단어를 삭제했는데 단어장에서는 없어지고 통계에는 남아 있으면 결과가 이상해졌다. 이런 상황이 몇 번 생겨서, remove 기능에서는 관련된 파일들을 같이 정리하도록 했다. reset 기능도 실수로 눌렀다가 다 날릴까 봐 YES를 입력해야만 실행되게 만들었다. 이런 부분들은 처음엔 별거 아닌 것처럼 보였는데, 실제로 써보면 꽤 중요한 부분이었다.

전체적으로 보면 아주 복잡한 프로그램은 아니다. 기능도 많지 않은 편이다. 그래도 기획부터 만들고, 중간에 고치고, 인공지능의 힘도 빌려보고, 다시 수정하는 과정까지 한 번에 해봤다는 점에서는 의미가 있었다. 다음에는 자주 틀리는 단어만 따로 모아서 다시 퀴즈를 내보는 기능을 넣어보고 싶다. 작은 프로그램이지만, 파이썬을 문법으로만 보지 않고 실제로 뭔가를 만드는 도구로 써봤다는 점에서 개인적으로는 꽤 의미 있는 경험이었다.
